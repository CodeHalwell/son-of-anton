## 2024-05-24 - Cross-Site Scripting (XSS) via DOM attribute injection in Trace Viewer Webview
**Vulnerability:** XSS risk due to injecting unescaped trace data (`span.id`, `span.type`) into HTML string concatenations rendered within `TraceViewerPanel.ts` and relying on the DOM node property `div.innerHTML` inside the custom `escapeHtml` function. Since `div.textContent` escapes basic entities but doesn't escape double quotes (`"`), this allows an attacker providing a crafted `span.id` or `span.name` to perform attribute injection (`" onmouseover="..."`) and execute arbitrary JavaScript.
**Learning:** `div.innerHTML` serialization does NOT escape double quotes when generated from `div.textContent`. Using this method for an `escapeHtml` implementation is unsafe for preventing attribute injection when the output is concatenated within HTML attributes (like `<div data-span-id="${escapeHtml(span.id)}">`).
**Prevention:** Avoid DOM-based `innerHTML` escaping patterns. Instead, use a robust string replacement function that explicitly handles escaping quotes (`"`, `'`) alongside HTML brackets and ampersands.
