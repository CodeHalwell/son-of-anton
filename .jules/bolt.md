## 2024-05-20 - Set for O(1) in tight filter loops\n**Learning:** In operations dealing with UI state and ranges where one array filters another (e.g., cell selection logic looking up elements in `currentSelections` against a computed range `cellsInSelectionRange`), using `Array.prototype.includes()` introduces an O(N*M) algorithmic bottleneck. Converting the secondary array to a `Set` leverages `Set.prototype.has()` inside the filter, yielding a massive theoretical O(N+M) speedup, directly benefiting perceived UI responsiveness in large environments without sacrificing safety since `Set.has` uses the same reference equality check.\n**Action:** Audit and convert `filter()` calls checking inclusion against dynamically sized lists to initialize `new Set()` prior to the iteration.
